# Установка и удаление CA-сертификатов минцифры на Debian 13.

## Дисклеймер
Все действия выполняются на ваш страх и риск. Не копируйте бездумно команды. Сначала поймите что они делают и убедитесь, что это действие безопасно.

## Преамбула
**Важно!** Полная поддержка шифрования ГОСТ отсутствует в распространенных linux-дистрибутивах. Она работает только в некоторых российских. Для ее реализации необходимо устанавливать соответствующий криптопровайдер, например, CryptoPro и переписывать софт. Пока минцифры выпускает совместимые сертификаты, но все может изменится. 

Это руководство применимо только для установки совместимых с Debian 13 сертификатов. Работа в других системах не гарантируется, но, можно попробовать. Деструктивных действий данное руководство не требует.

Если у вас что-то не заработало, создайте [issue](https://github.com/Web-Dev-SK/rus-certs/issues), попробуем разобраться вместе.

Я использую **sudo** для запуска команд от рута. Вы можете использовать другой способ.

## Немного общей информации (необязательно)
#### Файл */etc/ssl/certs/ca-certificates.crt*
Самый важный файл. Именно здесь лежат все сертификаты.

#### Файл */etc/ca-certificates.conf*
**В данном руководстве этот файл не используется!**

В этом файле перечислены сертификаты, которые вы хотите использовать или игнорировать для установки в */etc/ssl/certs*. Он генерируется автоматически командой ***dpkg-reconfigure ca-certificates***. Сертификаты должны находится в */usr/share/ca-certificates*. 

#### Основные папки, содержащие сертификаты:
1. */etc/ssl/certs/* - содержит ссылки на сертификаты и файл *ca-certificates.crt*, упомянутый выше.
2. */usr/share/ca-certificates* - сертификаты, устанавливаемые из пакетов.
3. */usr/local/share/ca-certificates* - дополнительные сертификаты.

## Установка сертификатов
**Создаем каталог для сертификатов и временный:**
```bash
sudo mkdir -p /usr/local/share/ca-certificates/rus
mkdir ~/ca_temp && cd ~/ca_temp
```

**Загружаем архивы сертификатов с сайта минцифры:**

корневые:
```bash
wget https://gu-st.ru/content/lending/linux_russian_trusted_root_ca_pem.zip 
```
и выпускающие:
```bash
wget https://gu-st.ru/content/lending/russian_trusted_sub_ca_pem.zip
```

**Распаковываем:**
```bash
unzip '*.zip'
```

**Перемещаем необходимые файлы в папку для сертификатов:**
```bash
sudo mv !(*_gost_*).crt /usr/local/share/ca-certificates/rus/
```
Файлы, содержащие **_gost_**  в имени перемещены не будут, так как не поддерживаются.

**Выполняем обновление:**
```bash
sudo update-ca-certificates -f
```
-f очистит ссылки в */etc/ssl/certs/* и удалит битые, если они есть.

**Убираем за собой**
```bash
cd .. && rm -fd ca_temp
```

## Проверка
Некоторые браузеры уже поддерживают эти сертификаты, поэтому лучше всего проверять консольными утилитами.\
**Внимание!** Многие подобные инструкции советуют проверять сайты sberbаnk.ru, gosuslugi.ru и подобные. **Это не правильно!** Эти сайты подсовывают обычные сертификаты. Перед проверкой, откройте сайт в браузере и убедитесь, что его сертификат соответствует подписанному именно минцифры . В поле подписанта должно быть что-то вроде **Russian Trusted Sub CA**. \
Пример проверки: запрос тайла с сайта роскадастра при помощи curl

## Удаление
Удалите папку */usr/local/share/ca-certificates/rus* и выполните обновление сертификатов:
```bash
sudo rm -fd /usr/local/share/ca-certificates/rus
sudo update-ca-certificates -f
```
